# Claude Review Agent

A Python-based PR review agent that analyzes GitHub pull requests and generates structured review comments.

## Features

- ✅ GitHub API integration
- ✅ Automated code review with security issue detection
- ✅ Structured Markdown output
- ✅ CLI and GitHub Action support
- ✅ Rule-based analysis (no API key required)
- ✅ Optional Claude API integration for semantic analysis

## Installation

### Requirements

- Python 3.8+
- `requests` (optional, for advanced features)

### Setup

```bash
# Clone or download
git clone https://github.com/YOUR_USERNAME/claude-review-agent.git
cd claude-review-agent

# Make executable
chmod +x claude-review.py

# Optional: Install as command
ln -s $(pwd)/claude-review.py ~/.local/bin/claude-review
```

### Configuration

Set environment variables for enhanced features:

```bash
# GitHub API token (increases rate limit from 60 to 5000 req/hour)
export GITHUB_TOKEN="ghp_your_token_here"

# Claude API key (optional, enables semantic analysis)
export ANTHROPIC_API_KEY="sk-ant-your_key_here"
```

## Usage

### CLI

```bash
# Review a PR by URL
python claude-review.py --pr https://github.com/owner/repo/pull/123

# Short format
python claude-review.py --pr owner/repo/123

# Save to file
python claude-review.py --pr owner/repo/123 -o review.md

# JSON output
python claude-review.py --pr owner/repo/123 --format json

# Review a local diff
git diff main > changes.diff
python claude-review.py --diff changes.diff --repo owner/repo --pr-title "My changes"
```

### GitHub Actions

Include the workflow file in your `.github/workflows/` directory:

```yaml
name: PR Review Bot

on:
  pull_request:
    types: [opened, synchronize]

permissions:
  pull-requests: write
  contents: read

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Download Review Agent
        run: |
          curl -LO https://raw.githubusercontent.com/YOUR_USERNAME/claude-review-agent/main/claude-review.py
          chmod +x claude-review.py
      
      - name: Run Review
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          python claude-review.py \
            --pr ${{ github.repository }}/${{ github.event.pull_request.number }} \
            --output review.md
      
      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.md', 'utf8');
            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: review
            });
```

## Output Format

The agent generates structured Markdown:

```markdown
## 🤖 Automated PR Review

### Summary
**Fix authentication bug**

This PR modifies 3 files with 45 additions and 12 deletions.
Files: `auth.py`, `login.py`, `test_auth.py`.

### ⚠️ Identified Risks
- Possible hardcoded credential in `config.py`
- Potential SQL injection in `query.py` - use parameterized queries

### 💡 Improvement Suggestions
- Consider adding tests for the changes
- Resolve TODO/FIXME in `auth.py`

### 📊 Confidence: Medium

---
*Generated by Claude Review Agent*
```

## Security Checks

The agent detects:

- ✅ Hardcoded credentials (password, API keys, tokens)
- ✅ SQL injection patterns
- ✅ Command injection patterns
- ✅ TODO/FIXME comments
- ✅ Large PR warnings (suggest breaking up)
- ✅ Missing test coverage
- ✅ Missing documentation

## Customization

### Adding Custom Rules

Edit `analyze_diff()` in `claude-review.py`:

```python
# Add your pattern
if "your_pattern" in line.lower():
    risks.append("Your custom warning message")
```

### Using Claude API

Set `ANTHROPIC_API_KEY` for semantic analysis:

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
python claude-review.py --pr owner/repo/123
```

This enables deeper understanding of code context, not just pattern matching.

## Limitations

- Rule-based analysis may have false positives
- Diff truncated to 10,000 chars for Claude API
- Rate limited without GitHub token
- No syntax-aware analysis (token-based only)
